# 机动车尾气监管平台全栈 Web 应用开发

请开发一个“机动车尾气监管平台”全栈 Web 应用，系统包含前台用户端、后台管理端，并支持通过 Docker 一键部署。系统主要用于机动车尾气检测、车辆环保信息管理、检测站监管、数据统计分析和异常预警。

## 一、项目目标

构建一个面向环保监管部门、机动车检测站、企业/车主用户的综合监管平台，实现机动车尾气检测数据的采集、管理、查询、分析与监管。

系统应具备清晰的前后台结构，界面简洁、专业，适合政府监管、环保管理和检测业务场景使用。

## 二、核心特点

- 支持机动车基础信息管理
- 支持尾气检测记录管理
- 支持检测站信息管理
- 支持车辆环保状态查询
- 支持检测数据统计分析
- 支持超标车辆预警
- 支持监管部门后台审核与管理
- 支持不同角色登录和权限区分
- 支持前台用户查询车辆检测结果
- 支持后台管理人员维护数据
- 支持 Docker 部署，方便本地或服务器快速启动

## 三、系统角色

系统至少包含以下角色：

1. **平台管理员**
   - 管理用户、角色、权限
   - 管理检测站
   - 查看全平台统计数据
   - 处理异常检测记录

2. **监管人员**
   - 查看辖区车辆检测数据
   - 查看超标车辆列表
   - 审核检测站上报数据
   - 查看统计报表

3. **检测站工作人员**
   - 录入车辆检测数据
   - 管理本站检测记录
   - 查询车辆历史检测结果

4. **普通用户 / 车主**
   - 查询车辆环保状态
   - 查询车辆尾气检测记录
   - 查看检测是否合格

## 四、前台功能

前台面向普通用户和车主，主要功能包括：

- 首页概览
- 车辆环保信息查询
- 尾气检测结果查询
- 检测站信息查询
- 政策公告展示
- 用户登录 / 退出

查询方式可支持：

- 车牌号查询
- 车辆识别代号 VIN 查询
- 检测报告编号查询

## 五、后台功能

后台面向管理员、监管人员、检测站工作人员，主要功能包括：

- 数据看板
- 用户管理
- 角色权限管理
- 车辆信息管理
- 检测记录管理
- 检测站管理
- 超标车辆预警
- 检测数据审核
- 统计报表
- 公告管理
- 系统日志

数据看板可展示：

- 今日检测车辆数

- 超标率

## 十一、当前项目实现

本仓库已按需求创建全栈项目骨架，包含前台用户端、后台管理端、Spring Boot 后端服务、MySQL/Redis/Nginx/Docker Compose 部署配置。

### 目录结构

```text
.
├── backend/                 # Spring Boot 3 后端服务
│   ├── src/main/java/        # 控制器、服务、模型、配置
│   └── src/main/resources/   # 应用配置与数据库初始化脚本
├── frontend/                # Vue 3 + Vite + TypeScript 前端
│   └── src/
│       ├── api/             # Axios 接口封装
│       ├── router/          # 前台与后台路由
│       ├── stores/          # Pinia 登录状态
│       └── views/           # 首页、登录页、后台看板
├── nginx/                   # 统一入口反向代理配置
├── docker-compose.yml       # 一键部署编排
├── .env.example             # 环境变量示例
└── .gitignore
```

### 已包含页面

- 前台首页：车辆环保状态查询、公告列表、检测站信息。
- 登录页：支持内置测试账号登录。
- 后台管理端：数据看板、检测趋势、排放标准占比、检测记录审核、超标车辆预警。

| GET | `/api/inspections` | 查询检测记录 |

### 本地开发

前端开发：

```bash
cd frontend
npm install
npm run dev
```

后端开发：

```bash
cd backend
mvn spring-boot:run
```

前端默认运行在 `http://localhost:5173`，后端默认运行在 `http://localhost:8080/api`。前端 Vite 已配置 `/api` 代理。

### Docker 一键启动

```bash
docker compose up --build
```

启动后访问：

- 前台：`http://localhost`
- 后台：`http://localhost/admin`
- 接口：`http://localhost/api`
